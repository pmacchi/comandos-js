Cómo Subir Actualizaciones a GitHub

Sigue estos pasos para subir tus actualizaciones desde un repositorio local a GitHub usando la consola de Git:

  1. Abre la consola o terminal en tu sistema.
  2. Navega al directorio de tu repositorio local usando el comando cd: 
    cd ruta/al/repositorio
  3. Verifica el estado de tu repositorio para ver los cambios realizados: 
    git status
  4. Añade los cambios al área de preparación (staging area): 
    git add .
    Para archivos específicos, usa: 
    git add archivo1 archivo2
  5. Confirma los cambios con un mensaje descriptivo: 
    git commit -m "Tu mensaje descriptivo sobre los cambios"
  6. Asegúrate de estar en la rama correcta. Para ver la rama actual: 
    git branch
    Para cambiar de rama, usa: 
    git checkout nombre-de-la-rama
  7. Actualiza tu repositorio remoto en GitHub: 
    git push origin nombre-de-la-rama
    Reemplaza nombre-de-la-rama con el nombre de tu rama, por ejemplo, main o master.
  8. Verifica el resultado. Revisa tu repositorio en GitHub para asegurarte de que los cambios se hayan reflejado correctamente.

¡Eso es todo! Siguiendo estos pasos deberías poder subir tus actualizaciones a GitHub sin problemas. Levanta el localhost con browser-sync start --server --files "."
